# Саммари по ТЗ

Документ фиксирует текущее состояние коробочного решения `Семицветик` относительно ТЗ из `tz-semicvetik.docx`.

## 1. Что сейчас есть

Собрано и выложено рабочее коробочное решение на:

- `Flask + SQLite + Jinja2`
- `gunicorn + systemd + nginx`
- SSR-страницы без фронтенд-фреймворка
- хранение контента и заявок в SQLite

Публично доступно:

- `https://cvetik.analystexe.ru/podgotovka-k-shkole/`

## 2. Что сделано по ТЗ

### 2.1. Архитектура

Сделано:

- новый проект поднят на целевом стеке `Flask + SQLite + Jinja2`
- коробка вынесена отдельно от старого Django-сайта
- проект развёрнут как отдельный сервис `cvetik-box.service`
- старый сайт сохранён на сервере, но не выведен в публичный трафик

### 2.2. Модель данных

Сделано:

- `pages`
- `programs`
- `teachers`
- `schedule_slots`
- `program_subjects`
- `reviews`
- `leads`
- `events`
- `site_settings`
- отдельная таблица для админ-пользователя

Что важно:

- данные программ, страниц и посадочных блоков хранятся в JSON-полях
- лиды пишутся в SQLite
- проект уже seeded демо-данными

### 2.3. Страницы сайта

Сделано:

- главная
- лендинги программ
- хаб программ
- педагоги
- о центре
- цены
- мероприятия
- контакты
- блоговая статья

Покрытие близко к карте страниц из ТЗ, но в демо сейчас заполнен основной набор страниц и маршрутов, а не полный production-контент по всем 12 страницам вручную.

### 2.4. Компонентная система

Сделано:

- страницы рендерятся из массива блоков в БД
- реализован компонентный рендер через `render_block.html`
- можно менять порядок блоков и данные без правки Python-кода

Реализованные компоненты:

- `hero`
- `trust_bar`
- `form_strip`
- `programs_grid`
- `advantages`
- `results_banner`
- `teachers_grid`
- `reviews_grid`
- `how_it_works`
- `faq`
- `timeline`
- `compare_table`
- `cta_final`
- `gallery`
- `events_grid`
- `articles_grid`
- `prices_table`
- `contact_panel`
- `rich_text`

### 2.5. Админка

Сделано:

- рабочая админка на `/admin/`
- вход через отдельный логин
- CRUD по ключевым сущностям через `Flask-Admin`
- можно редактировать страницы, программы, педагогов, отзывы, события, заявки, настройки

Ограничение:

- управление блоками страниц сейчас через JSON-поля, а не через визуальный drag-and-drop редактор

### 2.6. Формы и лиды

Сделано:

- формы на ключевых экранах
- AJAX POST на `/api/lead`
- нормализация телефона
- honeypot-поле
- rate limit на endpoint
- сохранение `source_page` и `source_block`
- запись лида в SQLite

Проверено:

- боевой лид через домен успешно создаётся

### 2.7. SEO

Сделано:

- SSR
- `title` и `meta description`
- `Open Graph`
- `schema.org`
- `robots.txt`
- `sitemap.xml`
- канонические URL

### 2.8. Деплой

Сделано:

- отдельный прод-каталог `/srv/cvetik-box`
- отдельный Unix socket
- отдельный `systemd`-юнит
- `nginx` переключен на новый сайт
- HTTPS уже работает через существующий сертификат домена

## 3. Что сделано частично

### 3.1. Телеграм-уведомления

Сделано частично:

- код отправки в Telegram есть
- интеграция поддерживается через `.env`

Не доведено:

- боевые `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` не подключены

### 3.2. Контент и наполнение

Сделано частично:

- демо-контент, демо-программы, демо-педагоги, демо-отзывы, демо-статьи есть

Не доведено:

- реальные тексты, реальные цены, реальные фото, реальные мероприятия, реальные контакты не внесены полностью

### 3.3. Мониторинг

Сделано частично:

- endpoint `/metrics` есть
- подготовлены docker-файлы и grafana/prometheus-шаблоны в репозитории

Не доведено:

- на этом сервере monitoring stack не поднят
- Docker на сервер не ставился
- Grafana/Prometheus/Alertmanager в бою не развернуты

### 3.4. Безопасность

Сделано частично:

- сайт работает за `nginx`
- HTTPS уже используется
- endpoint `/metrics` закрыт снаружи
- есть базовый rate limit на лиды

Не доведено:

- не настроены CSP headers
- не настроен fail2ban под этот сайт
- не настроен production backup-rotation для SQLite
- нет 2FA для админки

## 4. Что не сделано по ТЗ

Не сделано или не доведено до production-уровня:

- Alembic-миграции как полноценный управляемый слой обновлений коробки
- визуальный редактор блоков страниц
- загрузка фото педагогов через отдельный UX уровня CMS
- готовые Grafana dashboards в бою
- Alertmanager Telegram alerts
- Nginx exporter / Node exporter / Loki в проде
- автоматические бэкапы SQLite и статики
- CI/CD
- полный production security hardening из ТЗ
- полный набор страниц и контента в финальном коммерческом виде

## 5. Итог по этапам приёмки

### Этап 1. Скелет

Сделано в адаптированном виде:

- приложение работает
- БД есть
- маршруты и шаблоны работают
- nginx проксирует

Примечание:

- вместо Docker в проде применён более лёгкий стек `systemd + gunicorn + nginx`, потому что сервер слабый и Docker на нём не был установлен

### Этап 2. Данные и шаблоны

Сделано:

- модели есть
- seed есть
- Jinja2-шаблоны есть
- публичные страницы работают
- формы сохраняют лиды

### Этап 3. Админка

Сделано частично:

- CRUD есть
- вход и управление сущностями есть

Не доведено:

- визуальный редактор блоков и UX уровня mini-CMS

### Этап 4. Мониторинг и SEO

Сделано частично:

- SEO-основа есть
- `/metrics` есть

Не доведено:

- полный monitoring stack и alerts в проде

### Этап 5. Полировка и прод

Сделано частично:

- сайт выведен в сеть
- домен и HTTPS работают

Не доведено:

- финальный контент
- финальный дизайн
- продовые интеграции Telegram/Метрика
- бэкапы и эксплуатационный контур

## 6. Короткий честный вывод

Сейчас сделан не просто мокап, а рабочий коробочный прототип в сети:

- архитектура соответствует базовой идее ТЗ
- ключевой пользовательский путь работает
- сайт уже можно показывать, кликать и использовать как основу

Но это ещё не финальный production из ТЗ на 100%.

Текущее состояние точнее описывать так:

- `рабочий коробочный MVP / prototype in production`
- `готово для показа, обсуждения, наполнения и следующей итерации`
- `не завершён весь мониторинг, эксплуатация и контентный прод-уровень`
